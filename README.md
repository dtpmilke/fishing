# 🎣 Помощник рыболова

SPA на **Angular 22** без бэкенда: показывает атмосферное давление в выбранной точке карты (в мм рт. ст.) и прогноз клёва на основе его динамики. Дизайн в стиле Apple «liquid glass», светлая/тёмная/системная тема.

## Запуск

```bash
npm install      # один раз
npm start        # дев-сервер на http://localhost:4200
```

Сборка статики для деплоя:

```bash
npm run build    # результат в dist/fishing-helper/browser
```

### Деплой на GitHub Pages

`base href` уже относительный (`./`), а пути к service worker, манифесту и иконкам — тоже относительные, поэтому приложение работает в подпапке `/<repo>/` без правок.

```bash
npm run build
# залить содержимое dist/fishing-helper/browser в ветку gh-pages (например, через angular-cli-ghpages):
npx angular-cli-ghpages --dir=dist/fishing-helper/browser
```

Service worker регистрируется только не на `localhost` (чтобы не мешать `ng serve`), так что PWA и офлайн заработают именно на боевом домене.

## Как работает

- **Данные** — [Open-Meteo](https://open-meteo.com) (без ключа, CORS): `surface_pressure` и `pressure_msl` почасово за 2 дня назад и 2 вперёд. Гектопаскали переводятся в мм рт. ст. (`× 0.750062`). Ответ кэшируется на 1 час в `localStorage`.
- **Прогноз клёва** — балльная оценка (0–100) по тренду давления за сутки, стабильности (макс. почасовой скачок) и прогнозу на 12 ч. Стабильное/плавно падающее давление = хорошо, резкие скачки = плохо. Логика в `src/app/core/forecast.util.ts`, пороги в `config.ts`.
- **Выбор точки** — клик по карте (Leaflet), перетаскивание маркера, поиск по названию (геокодинг Open-Meteo), кнопка GPS, избранные места.
- **Без разрешения на геолокацию** — каскад: последняя точка → IP-геолокация (geojs/ipwho.is) → точка по умолчанию. Клик по карте работает всегда.
- **Темы** — светлая / тёмная / системная (`prefers-color-scheme`), выбор сохраняется. Карта: светлая = OSM (русские названия), тёмная = спутник; дефолтные контролы скрыты, есть лоадер тайлов.
- **Рыбные места рядом** — кнопка 🐟 ищет водоёмы и точки рыбалки через Overpass API (OpenStreetMap, без ключа) и ставит маркеры; тап по маркеру выбирает точку.
- **Осадки на карте** — кнопка дождя включает радарный слой RainViewer (без ключа). Ветер пока показан плиткой (слой ветра без ключа недоступен).
- **Bottom sheet** — на мобиле панель сворачивается/разворачивается перетаскиванием ручки (как в iOS); на десктопе — боковая панель.
- **PWA** — `manifest.webmanifest` + service worker (офлайн-кэш оболочки, приём пушей). Локальные уведомления работают сразу; для пуша при закрытом приложении нужен VAPID push-сервер (см. комментарий в `notification.service.ts`).

## Структура

```
src/app/
├── core/        # config, типы, утилиты давления/клёва, сервисы (weather, geo, theme)
├── state/       # app.store — состояние на signals
├── ui/          # map, search-box, verdict-card, pressure-chart, stats-row, theme-toggle
├── app.ts       # корневой компонент
└── app.html     # layout: карта + панель
```

## Производительность

Анимации только на `transform`/`opacity`; Leaflet на canvas (`preferCanvas`); zoneless + signals (точечная перерисовка); fallback на обычное размытие там, где нет `backdrop-filter`; уважается `prefers-reduced-motion`.

## Внешние сервисы (все бесплатные, без ключа)

Open-Meteo (погода + геокодинг), BigDataCloud (обратный геокодинг), geojs.io / ipwho.is (IP-локация), Esri / OpenTopoMap / CARTO (тайлы карты), Tabler Icons (иконки).

> Прогноз ориентировочный — это эвристика по давлению, а не гарантия улова.
